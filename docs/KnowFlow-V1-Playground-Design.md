# KnowFlow V1 Playground 设计说明

日期：2026-04-05

## 1. 目标

`/playground` 作为 `KnowFlow` 的内置联调控制台，服务两个阶段目标：

- 当前阶段：提升后端接口联调效率，减少手写 `curl` / Postman 反复切换的成本；
- 后续阶段：为更偏产品化的演示型聊天页提供可演进的视觉和交互底座。

第一版不追求独立前端工程，而是由 Go 后端直接托管静态页面，保证上手成本和维护成本最低。

## 2. 页面定位

页面不是传统后台表单，也不是纯聊天 Demo，而是“知识运营驾驶舱”：

- 左侧：工程控制面
- 右侧：聊天与结果展示面

这使页面既适合联调，也适合后续用于项目演示。

## 3. 路由与实现方式

- 入口路由：`GET /playground`
- 静态资源：
  - `GET /playground/assets/playground.css`
  - `GET /playground/assets/playground.js`

实现方式：

- 使用 Go `embed` 内嵌 HTML/CSS/JS
- 由后端直接返回页面和资源
- 页面只调用现有后端 API，不新增业务接口

## 4. 结构布局

整体采用“左控右显”双栏布局。

### 左侧控制栏

- 连接与上下文
  - `baseUrl`
  - `userId`
  - 当前 `sessionId`
  - 当前 `documentId`
  - 健康检查按钮
- 文档摄取
  - 文件上传
  - 原始文本录入
  - 上传结果摘要
- 知识操作
  - 知识反写输入
  - 重建索引
  - 最近一次工具执行反馈
- 会话导航
  - 刷新会话列表
  - 点击会话加载消息历史
  - 选中会话自动回填上下文

### 右侧主区域

- 顶部聊天输入区
  - 问题输入框
  - 普通问答按钮
  - SSE 问答按钮
  - 快捷问题 Chips
- 中部消息流
  - 用户消息
  - AI 回答
  - SSE 增量状态
  - citations 展示
- 底部调试抽屉
  - 原始响应 JSON
  - `retrieval_meta`
  - `tool_traces`
  - `/metrics` 快照

## 5. 视觉方向

采用“深色工作台 + 暖色知识纸张”的混合风格：

- 深石墨蓝背景，避免纯黑；
- 左栏偏仪表台质感，强调工程控制；
- 右侧回答卡片和 citation 卡片使用微暖纸张色，强调知识沉淀；
- 强调色仅保留青色与橙金色；
- 页面记忆点为“左边像工程控制台，右边像面试知识笔记本”。

## 6. 状态流

页面维护五类状态：

- `connectionState`
- `documentState`
- `chatState`
- `debugState`
- `sessionState`

关键自动化行为：

- 上传成功后自动回填 `documentId`
- 普通问答成功后自动回填 `sessionId`
- 点击会话列表后自动加载消息历史
- SSE 时先插入空 AI 消息，再持续拼接增量内容
- 每次请求都把原始响应同步到调试区

## 7. 首版范围

首版必须覆盖：

- `/playground` 单页入口
- 健康检查
- 文档上传
- 普通问答
- SSE 问答
- 会话列表
- 消息历史查看
- 知识反写
- 重建索引
- 指标查看
- 原始响应查看

首版不做：

- 登录鉴权
- 独立前端工程
- 实时图表大盘
- 产品化首页包装
- 复杂 Markdown 富文本渲染

## 8. 测试方式

实现完成后至少验证：

- `/playground` 页面可访问
- 页面可通过当前后端接口完成主链路联调
- SSE 输出可在页面中增量展示
- 操作反馈与错误提示清晰
- 桌面端体验优先，移动端至少可打开
